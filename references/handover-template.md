# Handover 模板

> 给 `writing-handovers` skill 用。**复制下面「核心骨架」到目标文件**，逐段填写；再按调用的 `type` 参照「type 附加字段」把额外内容织入对应段。`<...>` 是占位符，填写后删除尖括号。

---

## 核心骨架（复制这部分）

```markdown
---
type: experiment | task-chain | codedev | general
title: <工作线主题>
date: <YYYY-MM-DD>
focus_next: <下轮聚焦，来自调用参数；无则留空>
---

# Handover: <主题>

## 1. WHY — 目标/上下文
<这条工作线要达成什么、为什么做。给接棒者建立心智模型，2-4 句。>

## 2. WHAT — 本轮做了什么 + 产出
<本轮的行为与产出。大产物只引用路径/URL/commit，不复制内容。>
- <做了 X，产出在 `path/to/output`>
- <提交 `<commit hash>`：简述>

## 3. STATE — 当前状态
<什么已成立/已验证、什么还坏着或没做完。接棒者据此判断能信什么、不能信什么。>
- 已验证：<...>
- 未完成/已知坏：<...>

## 4. NEXT — 下轮 todo + 接棒须知
- [ ] <下一步具体动作>
- [ ] <...>

**接棒须知**：<建议调用的 skill / 已知坑 / 不要碰的东西>
```

---

## type 附加字段（按 type 织入对应段，不要把本节原样留在产出里）

| type | 在对应段额外织入 |
|---|---|
| **experiment** | WHY + 本轮**假设**；WHAT + **可复现步骤 & 关键参数**；STATE + **结果数据 & 结论可信度** |
| **task-chain** | STATE + **产出契约**（下游可依赖的输入/接口）；NEXT + **下游任务的明确入口** |
| **codedev** | WHAT + **改动文件清单 & diff 摘要**；STATE + **测试/构建命令 & 哪里还红**；NEXT + **如何验证** |
| **general** | 无附加，纯四段 |
